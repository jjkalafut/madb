Summary

Don’t you wish you could identify bad actor or malicious activity before it hits your network? MADB or the Malicious Activity Detection Bulletin is a dynamic graph anomaly detection analysis and visualization tech- nique that can be used to alert users to potentially malicious or unusual network activity. The program can accept your input and highlight po- tential “bad actors” or areas of risk within your network while provid- ing you with key details about your current network

Cyber attacks have cost the global economy over $1 Trillion USD in 2020. Most cybercrime ultimately involves retrieving or encrypting a user’s data. Malware, malicious software designed to infiltrate a device and/or network, is a key element, and most notably, it’s also used to distribute ransomware. Hospitals, Schools, Companies - they’ve all fallen victim to ei- ther losing their data or paying a hefty fee. Our solution focuses on helping users identify risker activity on their network, to help mitigate the risk of a potential attack.

Our solution leverages the MIDAS algorithm which is derived from a Chi- square goodness-of-fit test between the graph’s new structure brought in by the new incoming edges and the expected structure based on its cur- rent structure at a certain timestep. Here is the generalized formula:

We created thresholds for “abnormal” network traffic using a combina- tion of the output from the MIDAS algorithm with the principals from the Gaussian mixture model (GMM). Using this method, we were able to es- tablish a ROC-AUC score of 91.52% when identifying DoS, Scan11,and Scan44 attacks. Finally we allowed for a user to provide input on the total node risk scoring (TNS) by taking a weighted average between the algorithm output and the user input (with a default weighting of 60-40%, respectively). The Final Scoring is as follows:

TNS= (Algorithm Score* Weighting ) + (User Score* Weighting)

Data Set

We used the "UGR 16" data set for our analysis. This data set was developed by cybersecurity researchers to represent four months worth of network traffic. The "UGR 16" data set is approximately 50 GB and includes over external 600 million IP addresses, some of which are malware events. Given the scale and complexity of the "UGR 16" data set, substantial pre-processing was required to wrangle the data for further classification and analysis to identify malicious traffic.

Our Experiments and Approach

Attack Timing
Because our data included both normal and abnormal network data, volume of activity was not an accurate measure for “unusual” activity. We saw this when we looked at the edge and node distribution, so we made sure to run scoring across all timestamps for a particular “chunk” of data.

We tuned the MIDAS scoring algorithm over a number of “chunks” of data, for each we had to consider: • NormalCore: Basic MIDAS algorithm performed on each incoming edge
• RelationalCore: MIDAS algorithm performed on each incoming edge and the nodes related to it incorporating spatial and temporal relations
• FilteringCore: RelationalCore that allowed for a selection of a threshold to limit which new edges are allowed to affect suv based on their scores

In the end we decided on using the Relational Core with a 0.75 temporal factor and sum aggregation
We used GMM as a way to help establish thresholds for the color-coding portion of our solution, . We had to review the GMM model and learned that it is better able to model MIDAS’ scoring over larger periods of time. This resulted in our post- processing procedures having to run GMM overall all timestamps to get the best possible threshold. We also learned that MIDAS periodically misclassified certain anomalous behavior, GMM pushed these instances to the lower left during its model- ing, resulting in a need for manual adjustment to the thresholds.

Conclusions/Key Takeaways

We developed a network graph visualization to depict device traffic (by IP address) for each time "chunk" specified during data processing. The visualization is interactive and lets users toggle between time "chunks" to evaluate and understand the position of cyber threats. High risk devices are shown as large, red circles in relation to the other nodes. High network traffic volume is shown through comparatively thicker connection weights.
